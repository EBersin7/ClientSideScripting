Lab 1 - HTML Home Page with CSS

My experience with HTML, CSS, javascript/jquery, and client side frameworks before this class was just barely above zero. The farthest I had gone before with HTML was clicking 'View page source' on a website and even that was just an accident. The coding that I saw in HTML and javascript was familiar looking because of my prior CS classes, but it was still not something I believed I had a firm grasp on. Actual experience working in this field for this lab opened my eyes on the wide range of possibilities and control developers have over HTML documents.

I used several advanced techniques within this lab. One that immediately stands out on my page is the gradient effect on the content block. Casting it from white to light blue, it gives a look of gently fading color. Another special style I used was shadowing. By setting a color and pixel size, I was able to give both the header and the content block a small shadow effect underneath them. This helps create the illusion that the boxes are actually 3D and hovering over the background image.

I learned a lot in this lab, mainly everything a beginner needs to start work with HTML. One thing I picker up quickly was the use of tags in the language to create specialized sections. This led to research on the various tags that can be employed and what their general purpose is. The second big thing I learned was how a CSS works in the framework of a page. I was initially overwhelmed by the sheer number of options for design presented, but the naming conventions and common sense used in their functionality helped ease the learning process. Lastly, I learned how the angular directives and references allow for simple reuse of common code fragments. This makes designing a cohesive website much less work intensive.

Lab 2 - JavaScript Animation

The lab for this week involved using JavaScript to create a page with animation. Following up on my Super Mario theme, I took this opportunity to animate a sprite sheet of Mario himself from Super Mario Bros. 2. The Italian plumber can be made to run across the screen by pressing the "Move Mario" button on the animation page. When he reaches the green pipe, he disappears and reappears on the other end, true to his games. The other buttons on the page can be used to advance Mario frame by frame or to stop his run. Pressing the "Move Mario" button multiple times will increase his speed, but will require a page refresh to stop him.

I have previously taken several Java courses so I expected to be in familiar ground with JavaScript. However, there were a variety of differences which were surprising. The first big one is that variables did not need to have a type declared, they could accept and form to anything they were given. The other thing that I learned about was assigning buttons in the HTML to functions in the JavaScript. It was much simpler in JavaScript compared to Java which requires event handling, listeners, and GUI interaction. A link for this lab can be found here.

Lab 4 - Bootstrap Designed Page

In this lab, I created a newly designed page for my website's Mario theme. Using bootstrap, I was able to create a simple, but professional looking product sales page. The biggest point of this new design is that it is responsive to browser changes. The page can resize/reorder itself to suit mobile devices and larger screens smoothly. A specific component that is possible because of bootstrap is the row class. This class allowed me to create 3 separate panels for a product comparison side by side. The panels will stack on top of each other when the screen is shrunk. Another component that I used was the navbar. Design features in the navbar allow the links to become a drop down menu when the window is too small. This adds to the adaptability of the website design.

The layout I like best is the Blackbox. The page design incorporates many elements, but it keeps the text sections very clean. Colors used are vivid without being too flashy. Browser changes cause certain images within the page to bounce off the wall and shake. Standard switch action, like divs stacking on top of each other when shrunk are also present. Lab 7 - AngularJS Search

The lab this week invoked the web API we created last week to display information from our database in a sortable format. By using AngularJS, the table created on page load can be filtered by user input. Terms entered into the search box will limit the elements in the table to only those containing the search string. Each column of the table can be clicked on to sort the entire table by alphabetical order of that column. A consecutive click on the column name will reverse the order of the table based on the field. A small, unexpected (or possibly expected) result is that the search box will also look through the image URL to find a match. The only way I found to remedy this was to have a search box for each field. However, I found this to be too complex and ugly design-wise, so I left it simple and included instructions for use.
